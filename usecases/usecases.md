## UC-001. Создать новый vault
Пользователь выбирает папку или вводит имя нового vault-а.
1. Проверить путь.
2. Создать папку, если ее нет.
3. Создать запись vault-а в хранилище.
4. Создать локальное хранилище индекса (папка в appdata для служебных данных c index.db).
5. Сделать vault активным.
6. Поставить задачу первичного сканирования/indexing.

## UC-002. Подключить существующий vault
Пользователь выбирает уже существующую папку с .md файлами.
1. Проверить, что папка существует.
2. Проверить права на чтение/запись.
3. Зарегистрировать vault в приложении.
4. Создать для него локальное хранилище (папка в appdata c index.db).
5. Поставить задачу scan_vault.

## UC-004. Открыть vault
Пользователь выбирает vault из списка.
1. Проверить, что путь все еще существует.
2. Проверить доступность.
3. Сделать vault активным.
4. Запустить watcher файлов.
5. Вернуть metadata vault-а.
